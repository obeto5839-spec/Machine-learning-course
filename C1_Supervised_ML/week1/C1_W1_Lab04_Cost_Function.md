# C1_W1_Lab04 — 代价函数（Cost Function）

## 核心知识点

### 1. 代价函数的定义
代价函数衡量模型预测值与真实值之间的差距，用于评估模型好坏。

线性回归的代价函数（均方误差 MSE）：

$$J(w,b) = \frac{1}{2m} \sum_{i=0}^{m-1} (f_{w,b}(x^{(i)}) - y^{(i)})^2$$

其中 $f_{w,b}(x^{(i)}) = wx^{(i)} + b$

### 2. 分母为什么是 2m 而不是 m？
- 那个 **2** 是为了**求导方便**
- 平方项求导会产生系数 2，分母的 2 与之约掉，使梯度公式更简洁
- 常数倍缩放不影响最优解的位置

### 3. 代价函数的直觉理解
- 当 $w$ 和 $b$ 使预测完美匹配所有数据点时，$J = 0$
- 偏离最优参数越远，$J$ 增长越快（因为平方的作用）
- 在两点数据 $x=[1,2], y=[300,500]$ 的例子中，$w=200, b=100$ 时 $J=0$

### 4. 凸性（Convexity）
- 代价函数的误差表面是**凸的**（像碗底）
- 原因：平方函数是凸的，凸函数的非负加权求和仍然是凸的
- **好处**：只有一个全局最小值，没有局部最小值 → 梯度下降一定能收敛到全局最优

### 5. 数据不共线时
- 当数据点不在同一条直线上时，$J$ 的最小值 **不可能为 0**
- 因为线性模型只能画一条直线，无法同时穿过所有不共线的点

## 关键代码

```python
def compute_cost(x, y, w, b):
    m = x.shape[0]
    cost_sum = 0
    for i in range(m):
        f_wb = w * x[i] + b
        cost = (f_wb - y[i]) ** 2
        cost_sum = cost_sum + cost
    total_cost = (1 / (2 * m)) * cost_sum  # 注意是 cost_sum，不是 cost
    return total_cost
```

## 易错点
- 代码中最后求 `total_cost` 时要用累加后的 `cost_sum`，不要误用循环变量 `cost`（只是最后一个样本的误差）
- 分母 2 的作用不是"降维"，而是为了求导时约掉系数
